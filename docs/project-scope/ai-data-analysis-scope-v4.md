# AI 数据分析系统 - 需求大纲 V4.0

> 基于 V3.0 优化 | 优化日期: 2026-03-31
> 核心升级: AI 自动定义 ETL → 业务表单 → AI Skill 编排多层级处理 → 混合报表输出

---

## 1. 项目概述

**AI 数据分析系统** — 一个支持多数据源接入，AI 驱动全链路的数据 ETL 流水线平台。用户可通过 4 种方式定义业务数据，由 AI 自动设计 ETL 链路落库到业务表单，再通过 AI Skill 编排的多层级数据处理，最终输出图表+报告+交互追问的混合报表。

**核心价值:** 把"写 ETL 代码 / 配置数据流程"变成"说人话 + AI Skill 自动执行"

**与 V3.0 的核心区别:**

| 维度 | V3.0 | V4.0 |
|------|------|------|
| ETL 定义方式 | NL→SQL 查询为主 | AI 自动设计完整 ETL 链路（多层级），人确认后执行 |
| 数据存储 | 中间表/视图 | **业务表单**（承载业务语义） |
| 多层级处理 | NL 配置流程 | **AI Skill 编排** — 混合模式，支持全自动 + 人工辅助 |
| Agent 能力 | 无 | **多 Agent + Skill 配置平台**，支持自动分配 |
| 输出 | NL 查询 + 报表 | **混合输出** — 图表 + Markdown 报告 + 交互追问 |

## 5. 阶段一: 业务数据定义（4种模式）

### 5.1 纯对话模式

- **用户:** 业务人员
- **交互:** 用户用自然语言描述数据需求，AI 理解并生成 ETL 配置建议
- **示例:** "我想看本月各地区的销售情况"
- **AI 响应:** 自动解析需求 → 分析可用数据源 → 生成 ETL 链路 → 等待确认

### 5.2 表单 + AI 辅助模式

- **用户:** 数据分析师
- **交互:** 用户填写结构化表单（表名、字段、业务含义），AI 补充推荐映射关系
- **AI 辅助内容:** 字段业务语义推荐、数据源推荐、ETL 策略建议

### 5.3 双轨模式

- **用户:** 复杂组织，多类用户
- **交互:** 两种方式都可先用，系统智能判断走哪条路径
- **判断规则:** 复杂查询走对话，简单增删改走表单

### 5.4 模板 + AI 填充模式

- **用户:** 标准化行业场景
- **交互:** 管理员定义数据模板（如"销售数据模板""财务数据模板"），用户填空 + AI 校验
- **AI 校验:** 填写内容与模板匹配度检测、异常预警

## 6. 阶段二: AI 定义 ETL（核心新特性）

### 6.1 AI ETL 自动生成引擎

| 功能 | 描述 |
|------|------|
| **Schema 分析** | AI 自动解析数据源表结构（字段名、类型、主外键关系） |
| **ETL 链路设计** | AI 根据业务目标，自动设计完整 ETL 链路（包含数据抽取、清洗、转换逻辑） |
| **链路预览** | 生成 ETL 链路拓扑图，供用户预览 |
| **人工确认** | 用户确认链路设计后，AI 执行 ETL 任务 |
| **执行监控** | ETL 执行过程全程监控（抽取行数、耗时、错误） |

### 6.2 ETL 链路设计原则

- **下推优先:** 多数据源跨库 join 时，优先将查询下推到数据源执行（Federated Query）
- **视图优先，物化为辅:** 默认使用视图，仅在性能需要时物化中间结果
- **质量门禁前置:** ETL 每层完成后立即执行质量检查，不达标则阻断

### 6.3 质量门禁

| 功能 | 描述 |
|------|------|
| 内置测试用例 | 唯一性/非空/合法值/外键关系自动检测 |
| 质量评分 | 每层 ETL 后自动生成质量报告（缺失率/重复率/异常值） |
| 门禁阻断 | 评分低于阈值时流程中断，阻止脏数据进入下游 |
| 告警机制 | 质量异常时自动告警（邮件/钉钉/飞书/Webhook） |

## 7. 阶段三: 业务表单层

### 7.1 业务表单定义

业务表单是 ETL 结果的存储载体，同时承载业务语义，供下游 AI Skill 理解和使用。

| 功能 | 描述 |
|------|------|
| 表单设计器 | 定义表单结构（字段、类型、业务含义、标签） |
| 表单数据存储 | ETL 结果落库到业务表单，支持多种数据库 |
| 业务语义标注 | 字段标注业务语义（如：amount=销售额, region=地区） |
| 表单版本管理 | 表单结构变更记录版本，支持回滚 |

### 7.2 表单与 ETL 的绑定

- 每张业务表单在创建时绑定对应的 ETL 来源（数据源+抽取规则）
- 表单支持配置**自动刷新策略**（定时/事件触发）
- 表单数据变更自动记录审计日志

## 8. 阶段四: AI Skill 多层级处理（核心特性）

### 8.1 AI Skill 编排架构 — 混合模式

**层级化 Skill（外层框架）:**

| 层级 | 描述 | 内置原子 Skill |
|------|------|---------------|
| **清洗层 (DWD)** | 去重、格式标准化、空值处理、异常值过滤 | 数据清洗 Skill |
| **转换层 (DWT)** | 类型转换、字段映射、数据编码 | 数据转换 Skill |
| **聚合层 (DWS)** | 维度聚合、指标计算、多表关联 | 聚合计算 Skill |
| **应用层 (ADS)** | 最终数据输出到报表/数据产品 | 应用输出 Skill |

**混合模式:** 管理员定义处理层级框架，每层选用不同原子 Skill 组合。

### 8.2 原子化 Skill（可插拔）

每个 Skill 做单一具体任务，流水线自由编排:

| Skill 类型 | 职责 |
|-----------|------|
| 数据清洗 Skill | 去重、格式标准化、空值填充、异常值处理 |
| 数据过滤 Skill | 条件过滤、行级过滤、字段筛选 |
| 数据转换 Skill | 类型转换、编码转换、字符串处理 |
| 聚合计算 Skill | COUNT/SUM/AVG/排名、窗口函数、多维度聚合 |
| 指标计算 Skill | 业务指标计算（转化率/环比/同比/归因） |
| 关联处理 Skill | 多表 JOIN、UNION、lookup |
| 图表生成 Skill | ECharts 图表生成（折线/饼图/漏斗/地图等） |
| 报告生成 Skill | Markdown 分析报告生成 |
| AI 追问 Skill | 交互式 NL 追问处理 |

### 8.3 AI Skill 市场

- **平台内置 Skill:** 通过 Spring AI Alibaba Agent Framework 实现
- **第三方 Skill 接口（预留）:** 支持接入外部 Agent/Skill 服务
- **Skill 配置:** 每个 Skill 支持自定义参数配置、输入/输出 schema 定义

## 9. Agent 平台

### 9.1 多 Agent 架构

支持配置多个 Agent，每个 Agent 可绑定不同的 Skill 组合:

| 功能 | 描述 |
|------|------|
| Agent 注册 | 管理员注册 Agent，定义其能力描述、标签、可用 Skill |
| Agent 绑定 Skill | 每个 Agent 可绑定 N 个 Skill（原子化 Skill 或组合 Skill） |
| Agent 能力定义 | 每个 Agent 有明确的能力描述（供自动匹配使用） |

**示例 Agent:**

| Agent 名称 | 绑定的 Skill | 能力描述 |
|-----------|-------------|---------|
| 清洗专家 Agent | 数据清洗 Skill + 数据过滤 Skill | 擅长数据清洗、去重、空值处理 |
| 聚合分析 Agent | 聚合计算 Skill + 指标计算 Skill + 关联处理 Skill | 擅长多维度聚合、指标计算 |
| 报表大师 Agent | 图表生成 Skill + 报告生成 Skill + AI 追问 Skill | 擅长报表生成、报告撰写、交互分析 |
| 全链路 Agent | 全部 Skill | 端到端全流程处理 |

### 9.2 Agent 调度模式

| 模式 | 描述 |
|------|------|
| **手动选择** | 用户在配置多层级处理时，从 Agent 列表中手动选择使用的 Agent |
| **自动分配** | AI 根据 Agent 的能力定义（标签+规则+意图推断），自动匹配最合适的 Agent |

### 9.3 自动分配 — 混合匹配逻辑

自动分配优先级可配置，综合以下三种方式:

| 匹配方式 | 描述 |
|---------|------|
| **标签匹配** | Agent 和 Skill 打标签，表单/数据打对应标签，匹配标签对应的 Agent |
| **意图推断** | AI 分析任务内容（业务语义+数据特征），推断最合适的 Agent |
| **规则引擎** | 管理员配置匹配规则（如："包含销售字段 → 销售分析 Agent"） |

**混合优先级:** 管理员可配置优先级（标签 > 规则 > 意图推断，或自定义）

### 9.4 Agent 执行模式

| 模式 | 描述 |
|------|------|
| **全自动** | AI 自动完成全流程（ETL → 表单 → 多层级处理 → 报表），无需人工介入 |
| **AI 辅助，人工确认** | 每层处理 AI 建议，用户确认后执行 |
| **纯人工配置** | 用户完全手动配置处理链路，AI 仅提供推荐 |

## 11. AI 语义层（贯穿全链路）

### 11.1 业务术语映射

| 功能 | 描述 |
|------|------|
| 术语映射管理 | 业务术语与数据库字段的映射管理 |
| 多表歧义消解 | 知识图谱驱动的多表歧义消解（如"客户"在订单表/客户表/会员表中的字段映射） |
| 语义层版本化 | 术语映射变更记录版本，支持回溯 |

### 11.2 NL 理解与纠错

| 功能 | 描述 |
|------|------|
| NL 意图纠错 | 多轮对话确认查询意图（置信度 < 阈值时主动追问） |
| 查询置信度 | 显示 AI 生成 SQL/ETL 的置信度评分，执行前让用户确认 |
| SQL/ETL 修正 | 支持用户修正 AI 生成的链路/配置，修正历史记录 |
| 语义记忆库 | 用户修正时，AI 自动提取规则写入"机构知识库"，同类任务自动应用 |

### 11.3 RAG 语义检索

- NL 查询时，AI 自动检索相关历史修正记录（而非仅靠术语映射）
- 取相关性最高的 Top-K 作为上下文，提升歧义消解准确率

### 11.4 Intent Graph 意图图谱

- 建立用户意图层级结构（如"销售分析"下有"区域对比""趋势分析""归因分析"）
- AI 自动映射 NL 到意图树
- 区分探索性查询（无固定格式）和标准化查询（命中模板，走缓存加速）

## 16. 分析方式

| 分析类型 | 描述 |
|---------|------|
| **趋势分析** | 时间维度上的数据走势 |
| **对比分析** | 多维度数据对比（环比/同比/分组对比） |

## 17. 不做的事（边界）

- ❌ 不做复杂机器学习建模（预测分析仅限简单统计预测）
- ❌ 不做数据治理/全公司级元数据治理（仅做当前系统的数据字典和语义层）
- ❌ 不做实时流式数据处理（仅支持批量 ETL，架构预留事件驱动扩展能力）
- ❌ 不做复杂数据质量规则引擎（基础清洗过滤为主，复杂规则通过 Connector SDK 扩展）
- ❌ SSO 集成不含自定义 UI 改造（仅支持标准 SAML/OIDC 协议对接）
- ❌ 不内置第三方 Agent 实现（仅预留接口，第三方接入需单独集成）
