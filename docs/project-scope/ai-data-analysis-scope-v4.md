# AI 数据分析系统 - 需求大纲 V4.0

> 基于 V3.0 优化 | 优化日期: 2026-03-31
> 核心升级: 统一多层级 ETL 架构，数据同步与数据处理共用架构；数据源到业务表单支持 AI 辅助 ETL（含业务语义分析）；业务表单增加行级权限控制

---

## 1. 项目概述

**AI 数据分析系统** — 一个支持多数据源接入，AI 驱动全链路的数据 ETL 流水线平台。用户可通过自然语言或表单定义业务数据，由 AI 自动设计 ETL 链路落库到业务表单（支持行级权限控制），再通过统一的多层级数据处理架构，最终输出图表+报告+交互追问的混合报表。

**核心价值:** 把"写 ETL 代码 / 配置数据流程"变成"说人话 + AI Skill 自动执行"

**与 V3.0 的核心区别:**

| 维度 | V3.0 | V4.0 |
|------|------|------|
| ETL 架构 | 分场景独立设计 | **统一多层级 ETL 架构**，数据同步与数据处理共用 |
| 同步阶段 | 简单的抽取+映射 | **AI 辅助 ETL**（含业务语义分析） |
| 数据存储 | 中间表/视图 | **业务表单**（承载业务语义 + 行级权限） |
| 多层级处理 | NL 配置流程 | **AI Skill 编排** — 混合模式，支持全自动 + 人工辅助 |
| Agent 能力 | 无 | **多 Agent + Skill 配置平台**，支持自动分配 |
| 输出 | NL 查询 + 报表 | **混合输出** — 图表 + Markdown 报告 + 交互追问 |

## 3. 统一多层级 ETL 处理架构

### 3.1 两大应用场景

本系统采用**统一的多层级 ETL 处理架构**，适用于数据同步和数据处理两大场景：

| 场景 | 数据源 | 输出 | 说明 |
|------|--------|------|------|
| **场景1: 数据同步** | 第三方系统数据源 | 业务数据表 | 抽取+清洗+映射→业务表 |
| **场景2: 数据处理** | 业务数据表 | AI报表/分析报告 | 多层级处理→报表/报告 |

**两场景共用架构，仅数据源和输出不同。**

### 3.2 多层级结构

```
[数据源] → L0原始层 → L1清洗层 → L2聚合层 → L3报表层 → [输出:业务表 or 报表/报告]
            ↑          ↑          ↑          ↑              ↑
         AI辅助      AI辅助     AI辅助     AI辅助         AI生成
         抽取规则    清洗规则    聚合规则   报表规则       报表/报告
```

| 层级 | 处理内容 | AI 辅助能力 |
|------|---------|-------------|
| **L0 原始层** | 数据抽取、增量/全量标识、时间戳 | AI 自动分析源表结构，推荐抽取字段 |
| **L1 清洗层** | 去重、空值处理、格式标准化、异常值过滤 | AI 分析数据分布，自动生成清洗规则 |
| **L2 聚合层** | 维度聚合、指标计算（sum/avg/count）、窗口函数 | AI 理解业务语义，推荐聚合维度和指标 |
| **L3 报表层** | 最终数据集，关联语义模型指标 | AI 生成报表 SQL，自动匹配图表类型 |

### 3.3 完整数据流

```
第三方数据源 ──→ [AI辅助ETL同步] ──→ [业务数据表] (行级权限)
                                              │
                                              ▼
                                    ┌─────────────────┐
                                    │  业务语义层     │
                                    │ (术语映射/歧义消解)│
                                    └────────┬────────┘
                                             │
                                             ▼
    ┌──────────────────────────────────────────────────────────────┐
    │              AI Agent 多层级处理（混合模式）                    │
    └──────────────────────────────────────────────────────────────┘
                              │
          ┌───────────────────┼───────────────────┐
          ▼                   ▼                   ▼
     [L0原始层] ──→ [L1清洗层] ──→ [L2聚合层] ──→ [L3报表层]
          ↑              ↑              ↑              ↑
       AI自动生成    AI辅助定义    AI辅助定义     AI自动生成
          │              │              │              │
          └──────────────┴──────────────┴──────────────┘
                           │
                           ▼
                   [AI报表/分析报告]
```

## 5. 阶段二: AI 辅助 ETL 同步（数据源→业务表单）

### 5.1 AI 辅助 ETL 核心能力

| 功能 | 描述 |
|------|------|
| **Schema 分析** | AI 自动解析数据源表结构（字段名、类型、主外键关系） |
| **业务语义分析** | AI 分析字段业务含义，推荐字段映射关系（如：amt → 销售额） |
| **ETL 链路设计** | AI 根据业务目标，自动设计完整 ETL 链路（抽取规则、清洗规则、映射关系） |
| **链路预览** | 生成 ETL 链路拓扑图，供用户预览 |
| **人工确认** | 用户确认链路设计后，AI 执行 ETL 任务 |
| **执行监控** | ETL 执行过程全程监控（抽取行数、耗时、错误） |

### 5.2 AI 业务语义分析

| 功能 | 描述 |
|------|------|
| 字段语义识别 | AI 自动识别字段业务含义（金额/日期/状态/客户ID等） |
| 映射推荐 | 基于语义识别，推荐源字段到目标业务表单字段的映射关系 |
| 歧义消解 | 多表关联时，AI 自动消解同名字段的歧义（如"客户"在订单表/客户表中的不同含义） |
| 规则推断 | AI 从数据样本推断转换规则（日期格式统一、编码转换等） |

### 5.3 ETL 链路设计原则

- **下推优先:** 多数据源跨库 join 时，优先将查询下推到数据源执行（Federated Query）
- **视图优先，物化为辅:** 默认使用视图，仅在性能需要时物化中间结果
- **质量门禁前置:** ETL 每层完成后立即执行质量检查，不达标则阻断

### 5.4 质量门禁

| 功能 | 描述 |
|------|------|
| 内置测试用例 | 唯一性/非空/合法值/外键关系自动检测 |
| 质量评分 | 每层 ETL 后自动生成质量报告（缺失率/重复率/异常值） |
| 门禁阻断 | 评分低于阈值时流程中断，阻止脏数据进入下游 |
| 告警机制 | 质量异常时自动告警（邮件/钉钉/飞书/Webhook） |

## 6. 阶段三: 业务表单层

### 6.1 业务表单定义

业务表单是 ETL 结果的存储载体，同时承载业务语义，供下游 AI Skill 理解和使用。

| 功能 | 描述 |
|------|------|
| 表单设计器 | 定义表单结构（字段、类型、业务含义、标签） |
| 表单数据存储 | ETL 结果落库到业务表单，支持多种数据库 |
| 业务语义标注 | 字段标注业务语义（如：amount=销售额, region=地区） |
| 表单版本管理 | 表单结构变更记录版本，支持回滚 |

### 6.2 表单与 ETL 的绑定

- 每张业务表单在创建时绑定对应的 ETL 来源（数据源+抽取规则）
- 表单支持配置**自动刷新策略**（定时/事件触发）
- 表单数据变更自动记录审计日志

### 6.3 业务表单行级权限控制

| 功能 | 描述 |
|------|------|
| 权限规则配置 | 按用户属性（部门/区域/角色）配置过滤条件 |
| 权限继承 | 下级组织自动继承上级权限 |
| 权限生效时机 | 在业务表单层和应用层同时生效 |
| 权限审计 | 记录数据访问日志 |

**行级权限示例:**

| 用户角色 | 可查看数据范围 |
|---------|---------------|
| 区域经理 | 本区域数据（region = '华东'） |
| 部门主管 | 本部门数据（dept_id = 用户.dept_id） |
| 管理员 | 全部数据 |

## 7. 阶段四: AI Skill 多层级处理（核心特性）

### 7.1 多层级处理节点架构

多层级处理由多个节点组成，每个节点可独立选择执行 Agent：

```
[L0原始层节点] ──→ [L1清洗层节点] ──→ [L2聚合层节点] ──→ [L3报表层节点]
       │                  │                  │                  │
   Agent选择          Agent选择          Agent选择          Agent选择
```

**每个节点支持两种 Agent 选择模式:**

| 模式 | 描述 |
|------|------|
| **手动选择 Agent** | 用户从预设 Agent 列表中选择，每个 Agent 关联一个或多个 Skill |
| **AI 自动分配** | AI 根据 Agent 的能力描述（标签+意图推断），自动为节点分配合适的 Agent |

### 7.2 Agent 与 Skill 关联

| 功能 | 描述 |
|------|------|
| Agent 绑定 Skill | 每个 Agent 可绑定 N 个 Skill（原子化 Skill 或组合 Skill） |
| Agent 能力描述 | 每个 Agent 有明确的能力描述，供 AI 自动匹配和用户手动选择 |
| Skill 组合 | 多个原子 Skill 可组合成一个组合 Skill，绑定到单个 Agent |

**示例 Agent 及其关联 Skill:**

| Agent 名称 | 绑定的 Skill | 能力描述 |
|-----------|-------------|---------|
| 清洗专家 Agent | 数据清洗 Skill + 数据过滤 Skill | 擅长数据清洗、去重、空值处理 |
| 聚合分析 Agent | 聚合计算 Skill + 指标计算 Skill + 关联处理 Skill | 擅长多维度聚合、指标计算 |
| 报表大师 Agent | 图表生成 Skill + 报告生成 Skill + AI 追问 Skill | 擅长报表生成、报告撰写、交互分析 |
| 全链路 Agent | 全部 Skill | 端到端全流程处理 |

### 7.3 AI 自动分配机制

AI 根据 Agent 能力描述自动分配合适的 Agent：

| 匹配方式 | 描述 |
|---------|------|
| **标签匹配** | Agent 打标签，节点任务打对应标签，匹配标签对应的 Agent |
| **意图推断** | AI 分析节点任务内容（业务语义+数据特征），推断最合适的 Agent |
| **规则引擎** | 管理员配置匹配规则（如："清洗任务 → 清洗专家 Agent"） |

**混合优先级:** 管理员可配置优先级（标签 > 规则 > 意图推断，或自定义）

### 7.2 原子化 Skill（可插拔）

每个 Skill 做单一具体任务，流水线自由编排:

| Skill 类型 | 职责 |
|-----------|------|
| 数据清洗 Skill | 去重、格式标准化、空值填充、异常值处理 |
| 数据过滤 Skill | 条件过滤、行级过滤、字段筛选 |
| 数据转换 Skill | 类型转换、编码转换、字符串处理 |
| 聚合计算 Skill | COUNT/SUM/AVG/排名、窗口函数、多维度聚合 |
| 指标计算 Skill | 业务指标计算（转化率/环比/同比/归因） |
| 关联处理 Skill | 多表 JOIN、UNION、lookup |
| 图表生成 Skill | ECharts 图表生成（折线/饼图/漏斗/地图等） |
| 报告生成 Skill | Markdown 分析报告生成 |
| AI 追问 Skill | 交互式 NL 追问处理 |

### 7.4 Agent 执行模式

| 模式 | 描述 |
|------|------|
| **全自动** | AI 自动完成全流程（ETL → 表单 → 多层级处理 → 报表），无需人工介入 |
| **AI 辅助，人工确认** | 每层处理 AI 建议，用户确认后执行 |
| **纯人工配置** | 用户完全手动配置处理链路，AI 仅提供推荐 |

### 7.5 AI Skill 市场

- **平台内置 Skill:** 通过 Spring AI Alibaba Agent Framework 实现
- **第三方 Skill 接口（预留）:** 支持接入外部 Agent/Skill 服务
- **Skill 配置:** 每个 Skill 支持自定义参数配置、输入/输出 schema 定义

## 10. AI 语义层（贯穿全链路）

### 10.1 业务术语映射

| 功能 | 描述 |
|------|------|
| 术语映射管理 | 业务术语与数据库字段的映射管理 |
| 多表歧义消解 | 知识图谱驱动的多表歧义消解（如"客户"在订单表/客户表/会员表中的字段映射） |
| 语义层版本化 | 术语映射变更记录版本，支持回溯 |

### 10.2 NL 理解与纠错

| 功能 | 描述 |
|------|------|
| NL 意图纠错 | 多轮对话确认查询意图（置信度 < 阈值时主动追问） |
| 查询置信度 | 显示 AI 生成 SQL/ETL 的置信度评分，执行前让用户确认 |
| SQL/ETL 修正 | 支持用户修正 AI 生成的链路/配置，修正历史记录 |
| 语义记忆库 | 用户修正时，AI 自动提取规则写入"机构知识库"，同类任务自动应用 |

### 10.3 RAG 语义检索

- NL 查询时，AI 自动检索相关历史修正记录（而非仅靠术语映射）
- 取相关性最高的 Top-K 作为上下文，提升歧义消解准确率

### 10.4 Intent Graph 意图图谱

- 建立用户意图层级结构（如"销售分析"下有"区域对比""趋势分析""归因分析"）
- AI 自动映射 NL 到意图树
- 区分探索性查询（无固定格式）和标准化查询（命中模板，走缓存加速）

| AI 自动生成测试 | 根据数据分布自动推断异常规则并生成测试用例 | P1 |
| Data Contracts | 数据供应方与消费方签订契约，违反则阻断而非仅告警 | P1 |
| Schema 变更检测 | 上游表结构变更时自动告警，评估对下游流程的影响范围 | P2 |
